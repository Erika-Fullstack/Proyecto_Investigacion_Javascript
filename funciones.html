TIPOS DE FUNCIONES

Funciones Nominativas

1: Declaración de una función con nombre
        
                  // Declaramos una función con nombre
                  function saludar() {
                    console.log("¡Hola, mundo!");
                  }

                  // Llamamos a la función
                  saludar(); // Salida: ¡Hola, mundo!

La función es definida con el nombre "saludar"

2: Función con parámetros
        
                    function sumar(a, b) {
                      return a + b;
                    }
                    
                    const resultado = sumar(3, 7);
                    console.log(resultado); // Salida: 10

Las funciones pueden tener parámetros

3: Función que devuelve un mensaje personalizado
        
                  function presentar(nombre) {
                    return `Hola, me llamo ${nombre}.`;
                  }
                  
                  console.log(presentar("Carlos")); 
                  // Salida: Hola, me llamo Carlos.
                  console.log(presentar("Ana")); 
                  // Salida: Hola, me llamo Ana.
4:Función nominativa dentro de otra función
        
              function operacionMatematica(a, b, operacion) {
                return operacion(a, b); 
                // Llama a la función pasada como argumento
              }
              
              // Función nominativa para sumar
              function sumar(x, y) {
                return x + y;
              }
              
              // Función nominativa para multiplicar
              function multiplicar(x, y) {
                return x * y;
              }
              
              // Usamos las funciones nominativas dentro de otra función
              console.log(operacionMatematica(5, 3, sumar));       // 8
              console.log(operacionMatematica(5, 3, multiplicar)); // 15

Aquí hemos creado la función operacionMatematica que tiene 3 argumentos. Dos de ellos (a y b) son números y operación será una función, "sumar" o "restar". Definimos las funciones y después las llamamos como argumento.

5: Función nominativa dentro de un objeto
        

                    const calculadora = {
                      sumar: function(a, b) {
                          return a + b;
                      },
                      restar: function(a, b) {
                          return a - b;
                      }
                    };

                    console.log(calculadora.sumar(3, 5)); // Salida: 8
                    console.log(calculadora.restar(9, 4)); // Salida: 5

En este caso, los métodos sumar y restar son funciones nominativas asignadas como propiedades del objeto calculadora.

6: Función nominativa como métodos en clases
        
                          class Persona {
                            constructor(nombre) {
                                this.nombre = nombre;
                            }
                              
                            saludar() {
                                return `Hola, soy ${this.nombre}`;
                            }
                          }
                          
                          const persona = new Persona("Agatha");
                          console.log(persona.saludar()); 
                          // Salida: Hola, soy Agatha

En este caso, el método saludar es una función nominativa que forma parte de la instancia persona..

Funciones Anónimas

1: Función anónima básica en una variable
           
                            const suma = function (a, b) {
                            return a + b;
                          };
                          
                          console.log(suma(3, 4)); // 7

Aquí asignamos una función anónima a la variable suma, y luego la usamos como si fuera una función normal.

2: Función anónima en un evento
           
                          document.querySelector('button')
                          .addEventListener('click', function () {
                            alert('¡Botón clickeado!');
                          });

La función que se pasa como segundo argumento a addEventListener es anónima. Esta es una de las aplicaciones más comunes.

3: Función anónima como parámetro en un array
           
              const numeros = [1, 2, 3, 4, 5];

              // Usamos una función anónima dentro de forEach
              numeros.forEach(function (numero) {
                console.log(`El número es: ${numero}`);
              });  
              
              /* Aunque la manera más habitual es unar una función
                 flecha */
              numeros.forEach((elemento) => console.log(`El número es: ${elemento}`));

Este enfoque es común para realizar operaciones en cada elemento de un array sin transformar los datos

4: Función anónima con una función autoejecutada
           
          (function () {
            console.log('¡Esta es una función anónima autoejecutada!');
          })();

Esta es una función anónima que se ejecuta inmediatamente después de ser definida. Es útil para inicializar código sin contaminar el espacio global.

5: Función anónima con setTimeout
           
            setTimeout(function () {
              console.log('Este mensaje aparece después de 2 segundos');
            }, 2000);

Pasamos una función anónima como argumento a setTimeout.

6: Uso de funciones anónimas en "closures"
           
                    function contador() { 

                      let cuenta = 0; 
                      return function() { 
                             cuenta++; 
                             return cuenta; 
                             }; 
                      }
                      const incrementar = contador(); 

                      console.log(incrementar()); // Salida: 1 
                      console.log(incrementar()); // Salida: 2

Un closure (o clausura) en JavaScript es una función que "recuerda" el contexto en el que fue creada, incluso después de que dicho contexto haya terminado de ejecutarse.


En este caso, la función anónima devuelta por contador recuerda el valor de la variable cuenta, incluso después de que la función contador haya terminado de ejecutarse.
